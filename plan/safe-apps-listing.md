# Safe Apps catalog (optional follow-up)

Use this checklist when applying for inclusion in the official Safe Apps list (process and requirements change over time — verify against [Safe documentation](https://docs.safe.global/config-service-configuration/add-or-edit-safe-app)).

## Before you apply

- [ ] Production URL is stable (e.g. GitHub Pages with custom domain or long-lived path).
- [ ] [`public/manifest.json`](../public/manifest.json) loads over HTTPS with correct `name`, `description`, `iconPath` / icons (square SVG, sufficient size per current Safe guidance).
- [ ] App works as **custom Safe App** in Safe{Wallet} (smoke test row 4 in README).
- [ ] Short test plan and feature summary for Safe QA (what vault/silo actions exist).

## Submission

- Follow the current Safe process (form / GitHub / config service — see docs linked above).
- Attach or link: repository, deployment URL, manifest URL, changelog or release notes.

## Not in scope for the repo

Listing approval is owned by Safe; this file is only an internal checklist.
